5. REGLAS DE NEGOCIO PRINCIPALES

5.1 Regla de cierre automático
Un afiliado solo puede estar en estado COMPLETADO cuando:


carnet
 entregado = TRUE

acuse = recibido o validado

formulario = recibido o validado

no tiene incidencias abiertas
El sistema debe calcular este estado automáticamente.



5.2 Regla de cierre parcial
Si solo uno de los documentos está recibido:

estado general = cierre parcial



5.3 Regla de inmutabilidad de cierre
Si un afiliado está en estado COMPLETADO:
NO se permite:

cambiar estado

cambiar responsable

editar corte

eliminar documentos

modificar datos críticos
Solo se puede modificar mediante proceso de reapertura.



5.4 Regla de reapertura
Solo usuarios con rol autorizado pueden reabrir un caso.
Requisitos:

motivo obligatorio

registro en auditoría

fecha y usuario



5.5 Regla de transición de estados
El sistema debe validar transiciones válidas:
Permitidos:

pendiente → asignado

asignado → contactado

contactado → en ruta

en ruta → entregado

entregado → pendiente recepción
No permitidos:

completado → pendiente

pendiente → completado

no localizado → completado



5.6 Regla de documentos independientes
El sistema debe manejar:

Acuse

pendiente

recibido

validado

rechazado

Formulario

pendiente

recibido

validado

rechazado
Ambos procesos deben ser independientes.



5.7 Regla de validación documental
No se puede:

validar documento sin archivo

cerrar caso con documentos rechazados



5.8 Regla de duplicidad
No permitir:

duplicar afiliado por cédula + corte

asignación simultánea a múltiples responsables



5.9 Regla de asignación

Todo afiliado debe tener un responsable

No puede pasar a "en ruta" sin responsable



5.10 Regla de carga masiva
El sistema debe:

validar datos antes de importar

detectar duplicados

permitir vista previa

generar lote
